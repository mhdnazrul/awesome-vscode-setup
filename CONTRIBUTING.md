# Contributing

## How to Contribute

1. Fork the repository.
2. Create a feature branch: `git checkout -b feat/my-change`
3. Make your changes following the existing conventions.
4. Commit with a descriptive message.
5. Push and open a pull request.

## Commit Style

Use conventional commits: `type: scope — description`

Examples:
- `feat: snippets — add Java snippets`
- `fix: scripts — handle missing code CLI gracefully`
- `docs: installation — clarify Windows setup`
- `clean: profiles — remove duplicate settings`

## Before Submitting

- Verify JSON files are valid.
- Check Markdown links resolve.
- Keep changes scoped (one logical change per commit).
- Do not commit machine-specific paths or secrets.

## Code of Conduct

This project adheres to the [Contributor Covenant](CODE_OF_CONDUCT.md). By participating, you agree to uphold its terms.
